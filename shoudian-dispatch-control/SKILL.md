# 受电方案调度控制

## 一、定位

本技能是受电方案生成的**总调度器**：识别意图 → 返回任务列表 → 串行调度子智能体 → 汇总供电答复单。

子智能体执行顺序（严格串行，禁止并行）：**勘察 → 接入 → 计费 → 计量**。

---

## 二、门禁执行顺序（严格串行，禁止并行）

全流程任务必须**依次**通过以下 4 个门禁。**每个门禁全部通过后，才能执行下一个门禁或下一步动作；任一项未完成，立即终止流程并输出错误信息。**

执行顺序：**门禁1 → 门禁2 → 勘察 → 接入 → 门禁3 → 计费 → 计量**（门禁4贯穿每个智能体调用前后）。

---

### 门禁 1：地图选点（最先执行）

在执行任何后续动作之前，逐项确认：

- [ ] 已输出 `// ###显示地图###` 标志？
- [ ] 已等待用户响应（点击地图选点 或 明确跳过）？
- [ ] 已解析响应：点击→提取经纬度；跳过且入参有经纬度→用入参值；跳过且入参无经纬度→**报错终止**？
- [ ] 已用选点经纬度更新入参的经度、纬度字段？

**以上全部打勾后 → 进入门禁2。任一项为否 → 立即终止流程。**

---

### 门禁 2：客户需求采集（门禁1通过后执行）

门禁1全部通过后，逐项确认：

- [ ] 已询问用户用电需求（如：尽快接电、高可靠性、经济性优先等）？
- [ ] 已等待用户输入诉求文本（不超时，一直等待）？
- [ ] 已记录诉求（仅用于方案比选，不传给任何子智能体）？

**以上全部打勾后 → 调用勘察智能体。任一项为否 → 立即终止流程。**

---

### 门禁 3：方案比选（接入智能体返回后执行）

接入智能体返回后，在调用计费子智能体之前，逐项确认：

- [ ] 已将接入智能体返回的方案展示给用户？
- [ ] 多套方案 → 已等待用户选择？单套方案 → 已等待用户确认？
- [ ] 已记录用户选定方案，用于后续计费/计量？

**以上全部打勾后 → 调用计费智能体。任一项为否 → 立即终止流程。禁止自动选择、禁止跳过比选。**

---

### 门禁 4：任务状态更新（贯穿每个智能体调用前后）

每个子智能体调用前后，逐项确认：

- [ ] 调用前已输出 `// ###调用智能体### {智能体名称} //`？
- [ ] 返回后已**原样输出**完整结果（禁止省略、改写、总结、用代码块包裹 JSON）？
- [ ] 已更新 TASKSETP 中对应步骤的 status，并重新输出更新后的任务列表？
- [ ] 失败时已按"失败处理"重试或终止？

**任一项为否 → 立即终止流程。**

---

## 三、执行流程

### 3.1 意图分类

| 用户意图 | 任务类型 | 调度 |
|----------|----------|------|
| 包含完整JSON / "生成方案" / "出方案" | **全流程** | 勘察→接入→计费→计量，走 4 个门禁 |
| 明确指定某环节 | **单项** | 仅对应单个智能体，不走红程门禁 |
| 纯概念提问 | **问答** | 无需调度，直接回答 |

### 3.2 全流程

```
意图识别 → 返回任务列表
  →【门禁1 地图选点】→【门禁2 需求采集】
  → 勘察(步骤1) → 合并勘察输出到入参
  → 接入(步骤2) →【门禁3 方案比选】
  → 计费(步骤3) → 合并计费输出到入参
  → 计量(步骤4) → 供电答复单
```

### 3.3 单项任务

意图识别 → 依赖校验（缺前置依赖时输出纯文本提示并终止，不自动补执行）→ 返回任务列表 → 调度执行 → 供电答复单。

单项任务**不执行**地图选点、需求采集、方案比选，直接使用入参中的经纬度。

---

## 四、输出格式（严格遵守）

**所有 JSON 一律直接输出纯文本，禁止用 ```json 代码块包裹。**

### 4.1 任务列表（意图识别后立即输出）

```
// ###本次任务需要调用###勘察子智能体、接入子智能体、计费子智能体、计量子智能体 //

📋 任务列表已生成：

// ###JSON_TASKSTEP_START###
{TASKSETP 模板，见 references/tasksetp-template.json}
// ###JSON_TASKSTEP_END###
```

TASKSETP 字段：`step`(序号)、`title`(方案名称)、`status`(未开始/进行中/已完成/失败)、`agent`(智能体名称)。

### 4.2 智能体调用标记与状态更新

调用前：
```
// ###调用智能体### {智能体名称} //
```

返回后：原样输出结果 → 更新 TASKSETP 对应步骤 status → 重新输出更新后的任务列表 → 执行下一步。

### 4.3 子智能体结果原样输出

**所有 JSON 一律直接输出纯文本，禁止用 ```json 代码块包裹。每个子智能体返回后，用对应标志原样包裹其完整 JSON 输出。**

> ⚠️ **输出段数不固定**：各子智能体返回的方案个数/段数是动态的，下表仅列出**可能出现的标志名**，模型必须按子智能体**实际返回的段数**逐段原样输出，**禁止自行增减段数、禁止按固定段数输出**。

| 子智能体 | 可能的标志名 |
|---------|-------------|
| 勘察 | `KANCHAFANGAN1` |
| 接入 | `JIERUFANGAN1` |
| 计费 | `JIFEIFANGAN1`、`JIFEIFANGAN2` |
| 计量 | `JILIANGFANGAN1`、`JILIANGFANGAN2`、`JILIANGFANGAN3` |

标志包裹格式（以勘察为例）：
```
// ###JSON_KANCHAFANGAN1_START###
{勘察结果 JSON}
// ###JSON_KANCHAFANGAN1_END###
```

> **门禁3 触发判据**：接入智能体返回后，检查 `比选方案` 数组长度。长度 > 1 → 展示全部方案供用户选择；长度 = 1 → 展示单套方案等待用户确认。确认前禁止调用计费智能体。

### 4.4 供电答复单（最终输出）

```
// ###JSON_GONGDIANDAFUDAN_START###
{供电答复单模板，见 references/power-supply-reply-template.json}
// ###JSON_GONGDIANDAFUDAN_END###
```

## 五、参数合并（步骤2、步骤4前执行）

合并后的完整参数传递给后续所有子智能体。

**调接入智能体前**（在勘察输出基础上合并）：将勘察输出的以下字段合并到入参，勘察输出优先、同名字段覆盖：
重要性等级、土地使用权获取时间、农业排灌设施、重点项目标签、保安负荷容量、是否影响电能质量、一级负荷容量、二级负荷容量。

**调计量智能体前**（在上述合并结果上再次合并）：从计费输出提取以下字段合并：
定价策略类型、需量核定值、基本电费计算方式、功率因数考核方式、电价码、峰谷执行标志、功率因数标准。

`Type` 字段仅用于展示，不参与参数合并。

---

## 六、失败处理（统一规则）

**所有调用失败统一处理：失败时重试 1 次，第 2 次失败立即终止整个流程。禁止编造、禁止跳过、禁止重试超过 1 次。**

| 情况 | 处理 |
|------|------|
| 第1次调用失败 | 重试1次（相同参数），提示 `⚠️ 调用失败，正在重试...` |
| 第2次调用失败 | 终止流程，输出 `❌ 调用失败（已重试），流程终止`，对应步骤 status 标"失败" |
| 服务不可用 / 工具不存在 | 立即终止流程 |
| 任一门禁未通过 | 立即终止流程，输出对应错误信息 |
| 缺必填字段 | 返回缺失字段列表，终止执行 |
| 意图模糊 | 追问：需要全流程还是某环节？ |

**禁止行为：** 第2次失败后自行思考回复 / 跳过调用继续执行 / 编造数据 / 用其他方式绕过调用。

---

## 七、最终输出：供电答复单

全部完成后汇总各智能体输出，生成供电答复单（格式见第四节 4.4）。失败智能体对应字段返回空字符串 `""`。

**字段来源映射：**

| 供电答复单字段 | 来源 |
|---------------|------|
| 是否具备供电条件 | 勘察智能体 |
| 用电户编号 / 工单编号 | 入参.申请单号 |
| 用电户名称 / 行业分类 / 负荷性质 | 入参直映 |
| 用电地址 | 入参.用能地址 |
| 重要电力用户 | 入参.重要性等级 |
| 需求类型 | 入参.业务类型 |
| 装接容量 | 入参.运行容量 |
| 供电容量 | 接入智能体（由其计算） |
| 业务费信息 | 计费智能体 |
| 客户接入系统方案 | 勘察 + 接入智能体 |
| 客户受电系统方案 / 接线简图 | 接入智能体 |
| 计量计费方案 | 计费 + 计量子智能体 |
| 项目概况 | 汇总各智能体输出 |

未返回字段留空（`""`）。联系人、联系电话、等级、电源路数等动态字段由智能体返回或用户补充。

---

## 八、入参

所有子智能体共用统一入参 JSON，由子智能体自行提取所需字段。必填字段与各业务场景示例见 `references/input-params-example*.json`。

**条件字段：**
- 经度/纬度：全流程任务必须通过门禁1地图选点获取或确认
- 负荷性质：高压客户（一级/二级/三级）
- 是否城镇规划范围内：高压客户
- 生产班次：大工业用电
- 重要性等级：部分场景
- 使用期限：临时用电
- 是否三零小微企业：低压非居民

`思考过程模式` 为子智能体条件参数，不影响调度流程。
